## 1. 배경

더투어샵(주)는 2005년 설립된 서울 기반 어드벤처 여행 전문 플랫폼으로,
스키/스노보드, 트래킹, 골프, 아웃도어 SIT 여행 및 MICE 단체 투어를 운영한다.

기존 견적 산출 방식은 **엑셀 수작업**으로 이루어졌으며 아래 문제가 있었다:
- 시즌별 단가 변동 반영 시 실수 빈번
- 여행지·상품별 원가 데이터가 분산 관리
- 담당자에 따라 견적 기준이 달라짐
- 고객 챗봇·자동 견적 시스템 연동 불가

이를 해결하기 위해 **LLM Wiki 기반 원가 관리 시스템**을 도입한다.

## ADR-001: LLM Wiki DB 구조 채택

### 결정
Karpathy LLM Wiki 패턴을 채택하여 더투어샵 지식 베이스를 구축한다.

### 배경
기존 RAG 방식은 질문마다 원본에서 재조합하여 지식이 누적되지 않음.
LLM Wiki 방식은 지식을 마크다운 파일로 축적·관리하여 복리처럼 쌓임.

### 구조

```
tourshop-wiki/
├── raw/          ← 원본 자료 (읽기 전용, 절대 수정 금지)
├── wiki/         ← LLM이 자동 생성·관리하는 지식 베이스
│   ├── index.md  ← 전체 목차 (LLM 자동 관리)
│   └── log.md   ← 작업 이력 (append-only)
└── CLAUDE.md     ← 전체 공통 규칙 정의
```

### 운영 방식
| 작업 | 설명 |
|------|------|
| Ingest | raw/ 에 파일 추가 → LLM이 wiki/ 자동 업데이트 |
| Query | wiki/ 기반으로 질문·답변 → 유용한 답변은 wiki/에 저장 |
| Lint | 주기적 위키 점검 (모순·오래된 정보·누락 교차참조) |

### 근거
- 컴퓨터 로컬 폴더 = 데이터베이스
- 마크다운 파일 = 데이터 레코드
- LLM = 데이터베이스 관리자
- 클라우드 서버·별도 인프라 불필요
- 향후 MCP 서버 연동, 앱/웹 서비스 확장 가능

## ADR-002: travelcost 원가 폴더 구조 설계

### 결정
여행 원가를 항목별 독립 폴더로 분리하고 각 폴더에 CLAUDE.md를 배치한다.

### 폴더 구조

```
raw/travelcost/
├── hotel-rates/          ← 호텔 숙박 원가
│   └── CLAUDE.md
├── transport-rates/      ← 교통 원가 (항공 제외)
│   └── CLAUDE.md
├── guide-fees/           ← 가이드·인솔자·강사 인건비
│   └── CLAUDE.md
├── lift_skipass-rates/   ← 리프트티켓·스키패스
│   └── CLAUDE.md
├── golf-rates/           ← 골프 (그린피·카트피·캐디피)
│   └── CLAUDE.md
├── activity-rates/       ← 액티비티 (수상·육상·어드벤처)
│   └── CLAUDE.md
├── insurance-rates/      ← 여행자·산악 보험
│   └── CLAUDE.md
├── airticket-rates/      ← 항공권 (FSC·LCC·전세기)
│   └── CLAUDE.md
└── entrance-fees/        ← 관광지 입장료
    └── CLAUDE.md
```

### 각 폴더 하위 구조
```
[원가항목]/
├── CLAUDE.md              ← 해당 원가 처리 규칙
└── [나라명]/
    └── [파일명]-[연도].md  ← 실제 원가 데이터
```

### 근거
- 원가 항목별 독립 관리로 담당자 역할 분리 가능
- 각 CLAUDE.md가 해당 폴더의 규칙서 역할
- 나라별 하위 폴더로 여행지 확장 용이
- 서브에이전트와 1:1 대응 구조

## ADR-003: 원가 데이터 룸당 요금 통일 기준

### 결정
모든 호텔 원가 데이터는 **룸당 요금**으로 통일한다.

### 배경
나라별로 호텔 요금 제공 방식이 다름:
- 일본: 1인당 요금 (per person) 방식 多
- 캐나다·유럽: 룸당 요금 (per room) 방식 多

### 통일 기준
| 원본 형식 | 변환 방식 |
|---------|---------|
| 룸당 요금 | 그대로 사용 |
| 1인당 요금 | 룸당 요금으로 환산 후 등록 |

**환산 공식**
```
1인실: 룸당 요금 = 1인당 요금 (그대로)
2인실: 룸당 요금 = 1인당 요금 × 2
3인실: 룸당 요금 = 1인당 요금 × 3
```

**원본 형식은 Notes에 기재**
예) `원본: 1인당 요금 → 룸당 환산`

### 근거
- 견적 산출 공식 통일 → 오류 감소
- 나라별 다른 형식 처리 로직 불필요
- 변수 최소화로 AI 견적 정확도 향상

## ADR-004: 서브에이전트 역할 분리 가이드라인

### 결정
원가 항목별로 전담 서브에이전트를 구성하고 역할 경계를 명확히 한다.

### 서브에이전트 구성

| 에이전트 | 담당 | 담당 안 하는 것 |
|---------|------|--------------|
| hotel-rates | 숙박 원가·식사포함 여부 | 교통·가이드 |
| transport-rates | 버스·셔틀·렌터카·기차 | 항공권 |
| guide-fees | 가이드·강사 인건비 | 인솔자 항공·숙박 |
| lift_skipass-rates | 리프트티켓 (2~6일권) | 렌탈·강습 |
| golf-rates | 그린피·카트피·캐디피 | 숙박·교통 |
| activity-rates | 수상·육상·어드벤처 액티비티 | 가이드·교통 |
| insurance-rates | 여행자·산악 보험 | 보험 가입 처리 |
| airticket-rates | FSC·LCC·전세기 항공권 | 공항세 임의 계산 |
| entrance-fees | 관광지 입장료 | 가이드·교통 |

### 공통 금지 사항 (전 에이전트)
- 원가 임의 추정 금지
- 통화 임의 환율 변환 금지
- 담당자 확인 없이 기존 파일 덮어쓰기 금지
- `[확인필요]` 항목은 반드시 목록으로 출력

### 개발 원칙
```
자동생성 → 수정 시도  (❌)
내가 정의 → AI가 구현  (✅)
```
각 에이전트 임무와 CLAUDE.md를 담당자가 직접 정의하고
실제 업무와 맞는지 하나씩 검증 후 적용한다.

### 근거
- 단일 범용 에이전트보다 전문 에이전트가 정확도 높음
- 역할 경계 명확화로 데이터 중복·충돌 방지
- 담당자별 관리 영역 분리 가능

## ADR-005: 날짜 기준 원가 데이터 통일

### 결정
모든 원가 데이터의 기준을 **실제 일자(YYYY-MM-DD)** 로 통일한다.

### 배경
원본 데이터가 다양한 형태로 제공됨:
- 시즌 코드 (A, B, C 등)
- 날짜 범위 (12월 말 ~ 1월 초)
- 주중/주말 구분
- 성수기/비수기 구분

### 통일 기준
| 원본 형식 | 변환 방식 |
|---------|---------|
| 시즌 코드 | 날짜별로 전개하여 실제 일자로 변환 |
| 날짜 범위 | 시작일~종료일 행 각각 생성 |
| 주중/주말 구분 | 해당 날짜에 각각 다른 요금 입력 |
| 성수기/비수기 | 해당 날짜에 각각 다른 요금 입력 |

**표준 테이블 형식**
```
| 날짜       | 항목1 | 항목2 | 비고 |
|------------|------|------|------|
| 2026-12-01 |      |      |      |
| 2026-12-02 |      |      |      |
```

### 적용 범위
호텔 / 리프트티켓 / 골프 / 액티비티 / 항공권 / 입장료 전 항목

### 근거
- 견적 시 출발일 하나로 모든 원가 자동 조회 가능
- 시즌 코드·주중주말 등 이중 변수 제거
- AI 견적 산출 오류 최소화
- 전 에이전트 데이터 조회 방식 통일

## ADR-006: 디렉터 에이전트 설계

### 결정
9개 서브에이전트를 총괄하는 디렉터 에이전트를 구성하고
임시 견적서 생성 → 담당자 확인 → 최종 발송 흐름을 표준화한다.

### 견적 요청 채널
B2C 고객 / B2B 여행사 / 내부 직원 모두 처리

### 견적 산출 모드
| 모드 | 설명 |
|------|------|
| 자동 견적 | 여행지 + 인원수 + 출발일 입력 시 자동 산출 |
| 항목별 선택 | 담당자가 포함 항목 직접 선택 후 산출 |

### 처리 흐름
```
요청 접수 → 입력값 수집 → 서브에이전트 순서대로 호출
→ 원가 합산 → 마진율 적용 → 임시 견적서 생성
→ 담당자 확인 → 최종 발송
```

### 서브에이전트 호출 순서
```
1. airticket-rates   5. golf-rates
2. hotel-rates       6. activity-rates
3. transport-rates   7. entrance-fees
4. guide-fees        8. lift_skipass-rates
                     9. insurance-rates (마지막)
```

### 핵심 규칙
- [확인필요] 항목 있는 상태로 최종 발송 금지
- 마진율 임의 적용 금지
- 담당자 승인 없이 고객 직접 발송 금지
