## 1. 시스템 개요

```
┌─────────────────────────────────────────────────────┐
│               더투어샵 LLM Wiki 시스템                │
│                                                     │
│  원본 자료  →  LLM Wiki DB  →  에이전트팀  →  서비스  │
│  (raw/)         (wiki/)        (18개)        앱·웹   │
└─────────────────────────────────────────────────────┘
```

**핵심 원칙**: 파일을 넣고 "ingest해줘" 한마디 → LLM이 지식 베이스 자동 구축

## 2. 비즈니스 구조도

### 고객 채널
```
                        더투어샵 (주)
                            │
          ┌─────────────────┼─────────────────┐
          │                 │                 │
        B2C               B2B               내부
     (일반 고객)         (여행사)           (직원)
          │                 │                 │
     앱·웹·챗봇         앱·웹·챗봇         앱·웹·챗봇
          │                 │                 │
    마진 자동 포함     B2C 마진 할인       원가만 산출
```

### 마진율 구조
```
원가 데이터
    │
    ├── 항목별 마진율 (hotel 15~20%, skipass 10% ...)
    │       ×
    └── 나라별 마진율 (일본, 캐나다, 키르기스스탄 ...)
            │
            ▼
    항목 + 나라 조합별 마진율
            │
    ┌───────┴───────┐
    B2C           B2B
   (100%)      (B2C에서 % 할인)
```

### 운영 여행지 × 상품
```
         스키  골프  트래킹  어드벤처  MICE
일본      ✓     ✓      ✓
캐나다    ✓            ✓      ✓
키르기스스탄         ✓      ✓
카자흐스탄          ✓      ✓
러시아              ✓      ✓
중국        ✓     ✓
유럽        ✓     ✓      ✓
국내                              ✓
```

## 5. 데이터 흐름도

```
[외부 원본 자료]
PDF / 엑셀 / 이미지 / URL / 텍스트
              │
              ▼
    [raw/ 폴더에 저장]
    해당 에이전트 CLAUDE.md 규칙 적용
              │
    "ingest해줘" 한마디
              │
              ▼
    [LLM이 자동 처리]
    ① 원본 분석·추출
    ② 표준 템플릿 변환
    ③ wiki/ 업데이트
    ④ index.md 갱신
    ⑤ log.md 기록
              │
              ▼
    [wiki/ 지식 베이스]
    항상 최신 상태 유지
              │
              ▼
    [견적·서비스 생성]
    디렉터 에이전트 → 서브에이전트 호출
              │
              ▼
    [고객·담당자 공유]
    B2C 앱 / B2B 포털 / 카카오톡 / 웹링크
```

## 6. 견적 플로우

```
고객 접속 (B2C / B2B / 내부)
        │
        ▼
① 여행 종류 선택
   🎿 스키  ⛳ 골프  🥾 트래킹  🏄 어드벤처  ✈️ 자유여행
        │
        ▼
② 나라 선택
   🇯🇵 일본  🇨🇦 캐나다  🇰🇬 키르기스스탄  🇰🇿 카자흐스탄 ...
        │
        ▼
③ 세부 항목 선택
   호텔 → 스키장/골프장 → 교통 → 가이드 → 보험 → 입장료
        │
        ▼
④ 기본 정보 입력
   출발일 · 여행 기간 · 인원수
        │
        ▼
⑤ 금액 자동 산출
   ┌─────────────┬──────────────┬──────────────┐
   │    B2C      │     B2B      │     내부      │
   │ 마진 자동   │ B2C에서      │  원가만       │
   │ 포함        │ % 할인       │  산출         │
   └─────────────┴──────────────┴──────────────┘
        │
        ▼
⑥ 결과 공유
   고객: 1인 단가 · 총액
   담당자: 원가 + 마진 전체 내역
```

## 7. 여행안내서 플로우

```
예약 확정
    │
    ▼
[출발 14일 전] 최초 생성
    ├── 예약 확정 내용
    ├── 여행 일정표
    ├── 여행지 정보
    ├── 기상예보 (14일치)
    ├── 위험요소 안내
    ├── 환율·환전 정보
    ├── 대사관·영사관
    └── 로컬 축제·행사
    │
    ▼
[출발 7일 전] 기상예보 갱신
    │
    ▼
[출발 전날] 기상예보 최종 갱신
    │
    ▼
고객 공유
    ├── 🌐 웹링크 (실시간 업데이트)
    ├── 📄 PDF 다운로드
    └── 💬 카카오톡 발송
```

## 10. 시작하기

### 전제 조건
- Claude Code 설치
- tourshop-wiki 폴더 생성

### 첫 번째 ingest
```bash
# Claude Code에서
cd tourshop-wiki
claude

# 원가 자료 넣고
"raw/travelcost/hotel-rates/japan/aomori-spring-2026.md 인제스트해줘"

# 질문하기
"일본 스키 10인 4박 견적 뽑아줘"
```

### 나라 추가 (유지보수 에이전트)
```bash
"베트남 추가해줘"
# → 전체 에이전트 + markup-guide 자동 업데이트
```
